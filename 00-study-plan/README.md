# 0. 스터디 플랜 & 진단

## 현재 수준 진단

### 자가 진단 체크리스트

| 항목 | ✅/❌ | 비고 |
|------|-------|------|
| 프로그래머스 Level 2 문제 70% 이상 풀 수 있다 | | |
| STL 컨테이너(vector, map, set) 자유롭게 사용 가능 | | |

## 일일 루틴

### 기본 루틴 (일 2~3시간)

```
1. [30분] 개념 학습 / 복습 노트 읽기
2. [40~60분] 새 문제 풀기 (타이머 필수!)
   - 못 풀면 → 풀이 확인 → 이해 → 다음날 재도전
3. [30분] 오답 복습 (2일 전 못 푼 문제 다시 풀기)
4. [20분] 풀이 정리 & 커밋
```

### 에빙하우스 복습 주기

| 복습 시점 | 행동 |
|----------|------|
| 당일 | 풀이 후 핵심 아이디어 메모 |
| 1일 후 | 메모만 보고 다시 풀기 |
| 3일 후 | 아무것도 안 보고 다시 풀기 |
| 7일 후 | 변형 문제 풀기 |
| 14일 후 | 해당 유형 종합 문제 |

### 문제 풀이 프로세스

```
1. 문제 읽기 (5분)
   → 입출력 예제 손으로 따라가기
   → N의 범위 확인 → 허용 시간복잡도 추정

2. 설계 (10분)
   → 어떤 알고리즘/자료구조?
   → 점화식 or 탐색 전략 정리
   → 엣지 케이스 미리 파악

3. 구현 (20~30분)
   → 설계대로 코딩
   → 예제 테스트

4. 디버깅 (10분)
   → 틀리면 엣지 케이스 확인
   → 반례 만들기

5. 회고 (5분)
   → 시간복잡도 맞았나?
   → 더 나은 풀이?
   → 실수한 부분 기록
```

## 약점 파악 & 보완 전략

### 보완 전략

| 상황 | 대응 |
|------|------|
| 특정 유형 정답률 < 30% | 해당 유형 1주 추가 집중 |
| 시간 초과가 잦음 | N 범위 → 시간복잡도 매핑 반복 학습 |
| 구현에서 자주 틀림 | 코너케이스 체크리스트 활용 |
| DP 점화식 못 세움 | 소규모 예제로 패턴 찾기 연습 |
| 그래프 문제 어려움 | 기본 BFS/DFS 50문제 풀기 |

### 레벨업 기준

- Level 3 문제를 **60분 내 50% 이상** 풀 수 있으면 → 다음 단계로
- 특정 유형 정답률 **30% 미만** → 해당 유형 1주 추가 집중
- **3회 연속** 모의고사에서 3문제 중 2문제 이상 풀면 → 목표 달성!
